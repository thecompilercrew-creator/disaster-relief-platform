## Reporting a Vulnerability

We take security issues seriously, especially because this platform may be used during **real-world emergency situations**.

If you discover a security vulnerability, **please do not open a public GitHub issue**.

### Responsible Disclosure Process

1. Email the project maintainers with:
   - A clear description of the vulnerability
   - Steps to reproduce
   - Potential impact
2. Allow reasonable time for investigation and patching
3. Once resolved, public disclosure can be coordinated

## Security Measures in Place

- Password hashing
- JWT-based authentication
- Expiring access tokens
- Controlled access to sensitive contact information
- Environment-based secret management

---

## Out of Scope

- Social engineering attacks
- Physical access attacks
- Attacks requiring compromised user devices

---

## Mission-Critical Note

Because this application may be used during disasters, **any vulnerability that risks user safety or privacy is treated as critical**.

Thank you for helping keep this project safe.
